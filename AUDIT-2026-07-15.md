# 🔴 CRITICAL

## C1 — `/api/seed` autentifikatsiyasiz va BARCHA apartamentlarni o'chiradi
- **Joy:** `src/app/api/seed/route.ts:5-21`
- **Sabab:** `export async function GET()` — hech qanday auth yo'q. Service-role kaliti bilan
  ishlaydi va birinchi amali — `apartments.delete().neq("id", <zero-uuid>)` — ya'ni **hamma qatorni o'chirish**.
- **Xavf:** Istalgan odam `https://asiaway.uz/api/seed` ni ochsa, **produksiya apartamentlari
  o'chib ketadi**. `apartment_images` CASCADE bilan o'chadi, `bookings.apartment_id` NULL bo'ladi
  → bronlar xonasiz qoladi, moliya hisoboti buziladi. Qayta tiklash faqat backup'dan.
- **Yechim:** Endpoint'ni **butunlay o'chirish** (seed bir marta ishlatilgan). Kerak bo'lsa —
  `SEED_SECRET` header + `NODE_ENV !== "production"` cheklovi.
- **Murakkablik:** Trivial (5 daq)

## C2 — Har qanday foydalanuvchi o'zini `shef` qila oladi (RLS privilege escalation)
- **Joy:** `supabase/migrations/20260712000000_dashboard_ideal.sql` — `CREATE POLICY "Admin All Profiles" ON profiles FOR ALL TO authenticated USING (true) WITH CHECK (true)`
- **Sabab:** `profiles` jadvalida **har qanday autentifikatsiyalangan foydalanuvchi** istalgan
  qatorni **UPDATE** qila oladi — jumladan o'zining `role` ustunini.
- **Xavf:** Farrosh (tozalovchi) brauzer konsolidan bitta so'rov bilan `role='shef'` qilib,
  moliya, egalarga to'lov, xodimlar ish haqi — hammasiga kiradi. **To'liq auth bypass.**
- **Yechim:**
  ```sql
  DROP POLICY IF EXISTS "Admin All Profiles" ON profiles;
  -- Har kim FAQAT o'z profilini o'qiydi
  CREATE POLICY "read own profile" ON profiles FOR SELECT TO authenticated USING (id = auth.uid());
  -- Rolni faqat service_role o'zgartiradi (dashboard emas). UPDATE policy umuman bermaymiz.
  ```
  Rol berish faqat Supabase panel yoki service-role skript orqali.
- **Murakkablik:** Kichik (30 daq)

## C3 — Server Action'larda ROL TEKSHIRUVI UMUMAN YO'Q
- **Joy:** `src/app/dashboard/**/actions.ts` (barchasi), `src/app/actions/*.ts`
- **Sabab:** Avtorizatsiya faqat `src/lib/supabase/middleware.ts` da — u **sahifa URL'ini**
  tekshiradi. Lekin Next.js Server Action'lar — bu **ochiq POST endpoint'lar**; ularni
  sahifaga kirmasdan to'g'ridan-to'g'ri chaqirish mumkin. RLS ham `TO authenticated USING(true)`
  bo'lgani uchun bazada ham to'siq yo'q.
- **Xavf:** Farrosh roli bilan kirgan xodim quyidagilarni chaqira oladi:
  `markLeasePaid` (egaga to'lov + xarajat yozish), `addExpense`/`deleteExpense`,
  `deletePayment` (kassa yozuvini o'chirish), `addStaff`/`toggleStaffActive`,
  `updateBookingStatus`, `deleteApartment`. **Moliyaviy firibgarlik va ma'lumot yo'q qilish.**
- **Yechim:** Har bir "shef-only" action boshida rol tekshiruvi:
  ```ts
  // src/lib/auth.ts
  export async function requireRole(...roles: string[]) {
    const supabase = await createClient();
    const { data: { user } } = await supabase.auth.getUser();
    if (!user) throw new Error("Unauthorized");
    const { data: p } = await supabase.from("profiles").select("role").eq("id", user.id).single();
    if (!p || !roles.includes(p.role)) throw new Error("Forbidden");
    return p.role;
  }
  ```
  + RLS'ni rolga bog'lash (helper `auth_role()` funksiya + policy'larda `auth_role() = 'shef'`).
- **Murakkablik:** O'rta (1 kun — ~20 ta action)

## C4 — Narxni mijoz belgilaydi → to'lov summasini manipulyatsiya qilish
- **Joy:** `src/app/actions/booking.ts:7-18, 60-75` + `src/app/api/payments/payme/route.ts:95-100`
- **Sabab:** `createBooking(input)` brauzerdan kelgan `total_price` va `deposit_amount` ni
  **tekshirmasдан** bazaga yozadi. Payme webhook esa to'langan summani **o'sha bazadagi
  `deposit_amount`** bilan solishtiradi (`usdToTiyin(booking.deposit_amount)`).
- **Xavf:** Hujumchi `deposit_amount: 0.01` bilan bron yaratadi → Payme'da 1 tiyin to'laydi →

  Shu bilan birga `total_price` ham soxta → moliya hisoboti va cheklar buziladi.
- **Yechim:** Narxni **faqat serverda** hisoblash:
  ```ts
  const { data: apt } = await supabase.from("apartments")
    .select("price_per_day, deposit_amount").eq("id", input.apartment_id).single();
  const nights = differenceInDays(checkOut, checkIn);
  const total_price = nights * apt.price_per_day;      // mijozdan OLINMAYDI
  const deposit_amount = apt.deposit_amount;            // mijozdan OLINMAYDI
  ```
  Input'dan `total_price`/`deposit_amount`/`nights` ni butunlay olib tashlash.
- **Murakkablik:** Kichik (2 soat)

## C5 — Telegram webhook autentifikatsiyasiz → soxta callback bilan bazani o'zgartirish
- **Joy:** `src/app/api/telegram/webhook/route.ts:48+`
- **Sabab:** `POST` handler kelgan so'rovning **Telegram'dan ekanini tekshirmaydi**

  ```ts
  const secret = req.headers.get("x-telegram-bot-api-secret-token");
  if (secret !== process.env.TELEGRAM_WEBHOOK_SECRET) return new Response("Unauthorized", { status: 401 });
  ```

- **Murakkablik:** Kichik (1 soat)

## C6 — Anon kalit bilan istalgan bron/lead kiritish (RLS `WITH CHECK (true)`)
- **Joy:** `20260707000000_init.sql` (`Public Insert Bookings`), `20260708000000_leads.sql` (`Public Insert Leads`)
- **Sabab:** `FOR INSERT WITH CHECK (true)` — ommaviy `anon` kalit (u brauzerда ochiq) bilan

  Server Action orqali **service-role** bilan yozsin (lead'да allaqachon shunday). RLS'да
  `WITH CHECK` shartlarini qo'yish (masalan `check_out > check_in`, `total_price >= 0`).
- **Murakkablik:** Kichik (1 soat)
